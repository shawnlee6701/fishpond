# P1 素材落位与验收标准

本文档说明第二批 P1 素材的处理稿位置、正式接入位置和验收标准。当前可用处理稿位于 `Design/processed_p1/`，原始下载文件保留在 `Design/` 根目录。

## 使用规则

- 正式接入 Godot 时，从 `Design/processed_p1/` 复制到下表的 `最终放置位置`。
- 不要直接引用 `Design/` 根目录原始图；原始图多为 DALL·E 输出尺寸，未裁切到游戏槽位。
- 复制到 `assets/...` 后让 Godot 自动生成 `.import` 文件；不要手写 `.import` 或 `.uid`。
- `contact_sheet_p1.png` 只用于验收预览，不需要复制到 `assets/`。

## 命名说明

- P1 prompt 与代码未来槽位使用 `buyer_transfer_placeholder.png`。
- 早期总清单里的 `buyer_avatar.png` 指同一类买家形象，但本项目接入时优先使用 `buyer_transfer_placeholder.png`，避免和 `scripts/after_contract_choice.gd` 里的槽位名不一致。

## 通用验收标准

- 尺寸必须与目标尺寸一致。
- PNG 必须有 alpha 通道，背景不能残留白底、灰边或白色矩形。
- 所有素材必须无文字、无数字、无水印、无签名。
- 主体不能贴边：独立图标至少保留 2 到 6 px 安全边，横幅插图至少保留 8 px 安全边。
- 鱼种图标必须保持横向轮廓，头尾完整，在目标小尺寸下仍能分辨鱼种体量差异。
- 下网方法图标必须在 150 x 150 下能一眼区分：手捞、稳捞、抽干。
- `catch_result_visual.png` 与 `settlement_visual.png` 必须保留宽幅构图，不能被裁成单个物件图标。
- `buyer_transfer_placeholder.png` 必须是单人买家形象，不带对话框、背景场景或额外 UI 框。
- 风格保持粗黑墨线、手绘笔触、乡镇鱼塘生意感；不要变成光滑矢量、3D、写实摄影或日式可爱风。

## 导入 Godot 后检查

1. 用 `sips -g pixelWidth -g pixelHeight <file>` 检查 `assets/...` 中实际文件尺寸。
2. 打开图片预览或 `Design/processed_p1/contact_sheet_p1.png` 检查透明底和主体完整性。
3. 接入 UI 后在 1080 x 1920、720 x 1280、540 x 960 三个尺寸下检查：
   - 鱼获列表图标不挤压文字；
   - 下网方法图不撑高卡片；
   - 鱼获/结算横幅不遮挡标题和账单；
   - 买家头像在转包弹窗内不贴边；
   - 所有透明底素材都没有白边。
