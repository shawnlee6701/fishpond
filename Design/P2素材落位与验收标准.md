# P2 素材落位与验收标准

本文档说明第三批 P2 素材的处理稿位置、正式接入位置和验收标准。当前可用处理稿位于 `Design/processed_p2/`，原始下载文件保留在 `Design/` 根目录。

## 使用规则

- 正式接入 Godot 时，从 `Design/processed_p2/` 复制到下表的 `最终放置位置`。
- 不要直接引用 `Design/` 根目录原始图；原始图多为 DALL·E 输出尺寸，未裁切到游戏槽位。
- `Design/pond_action_button.pn` 是原始下载文件名异常，正式文件名必须使用 `pond_action_button.png`。
- 复制到 `assets/...` 后让 Godot 自动生成 `.import` 文件；不要手写 `.import` 或 `.uid`。
- `contact_sheet_p2.png` 只用于验收预览，不需要复制到 `assets/`。

## 命名说明

- 验塘卡片按当前代码槽位使用 `inspection_method_card_observe.png`、`inspection_method_card_fish_finder.png`、`inspection_method_card_master.png`。
- 早期总清单里的 `inspection_card_observe.png`、`inspection_card_fishfinder.png`、`inspection_card_master.png` 是旧名，不建议继续使用。
- 盈亏徽章当前是一张 `profit_loss_badge.png`，由代码按赚/亏/平状态叠加文字或颜色；不是三张 `profit_badge_green/red/neutral.png`。
- `pond_action_button.pn` 只存在于原始下载目录，处理稿和正式落位都必须叫 `pond_action_button.png`。

## 通用验收标准

- 尺寸必须与目标尺寸一致；自适应类素材以本文处理稿尺寸作为九宫格源图。
- 透明底素材必须有 alpha 通道，背景不能残留白底、灰边或白色矩形。
- 不透明矩形素材必须铺满整个画布，不能有多余白边。
- 所有素材必须无文字、无数字、无水印、无签名；游戏中文字由 Godot 控件覆盖。
- 用于文字覆盖的区域必须干净，不能有高对比图案穿过主要文本区。
- 九宫格素材边框和角必须完整，拉伸后边框不能明显变粗、变扁或断裂。
- 按钮和徽章主体不能贴边，至少保留 2 到 6 px 安全边。
- 纹理素材 `bg_card_paper.png` 与 `bg_parchment.png` 后续接入前应做平铺检查；如果出现明显接缝，需要再做偏移修补。
- 风格保持粗黑墨线、手绘笔触、乡镇鱼塘生意感；不要变成光滑矢量、3D、写实摄影或现代极简 UI。

## 导入 Godot 后检查

1. 用 `sips -g pixelWidth -g pixelHeight <file>` 检查 `assets/...` 中实际文件尺寸。
2. 打开图片预览或 `Design/processed_p2/contact_sheet_p2.png` 检查裁切、透明底和主体完整性。
3. 接入 UI 后在 1080 x 1920、720 x 1280、540 x 960 三个尺寸下检查：
   - 验塘卡片文字不被纹理压住；
   - 弹窗背景在九宫格拉伸后边框稳定；
   - 报价、余额、状态条里的数字可读；
   - 操作卡/下网卡/结算卡拉伸自然；
   - 对话气泡和按钮没有白边；
   - 包塘记录卡片展开/收起后背景不破图。

## 当前批次备注

- `transfer_dialog_bg.png`、`owned_pond_card_bg.png`、`action_card_bg.png`、`settlement_card_bg.png`、`record_card_bg.png` 属于自适应素材，本次处理稿给出可落位的参考源图尺寸，正式接入时应使用 NinePatch/NineSlice。
- `offer_price_badge.png`、`balance_highlight_bg.png`、`status_box_bg.png` 是不透明横条，适合直接作为条形背景。
- `button_danger_confirm.png`、`button_accept.png`、`pond_action_button.png` 已处理为透明底按钮背景。
