# 《这塘我包了》

## 项目类型

Godot 4.6 / GDScript / 2D 竖屏 MVP

## 核心玩法简介

玩家每天选择一口鱼塘，通过验塘判断价值，决定是否承包。承包后可以选择转包、卖一网或自己捞。最终根据鱼获、成本和收入结算利润。目标是在有限信息下搏大鱼和鱼王。

## 当前已实现功能

- 主菜单、现金和天数展示
- 每日生成 3 口鱼塘
- 鱼塘详情与验塘工具
- 承包确认与最低周转资金校验
- 承包后的转包、卖一网、自己捞等选择
- 低成本、标准、全部抽干等作业方案
- 鱼获模拟、收入计算和结算展示
- 鱼王特殊结果展示
- 下一地方推进与当日状态重置
- 第一版中文 MVP 文案

## 如何运行项目

1. 安装 Godot 4.6。
2. 用 Godot 打开本项目目录。
3. 运行主场景：

```text
res://scenes/Main.tscn
```

也可以在项目根目录执行基础检查：

```bash
godot --version
godot --headless --path . --quit
```

## 数值调参台

项目内置一个本地网页调参器：

```bash
python3 -m http.server 8766
```

然后打开：

```text
http://127.0.0.1:8766/tools/balance-lab/
```

调参台会读取 `data/balance_rules.json`，支持调整关键参数、三类鱼塘毛 ROI 目标区间、批量跑策略模拟、查看 ROI/破产率/现金分位数，并导出修改后的 JSON。

## 主要目录说明

- `scenes/`：Godot 场景文件，包含主菜单、鱼塘列表、鱼塘详情、承包后选择和结算界面。
- `scripts/`：GDScript 逻辑代码，包含状态管理、界面路由、鱼塘生成、验塘、捕鱼模拟和结算。
- `data/`：玩法数值和配置表，例如初始现金、作业成本、工具、鱼类和鱼塘类型。
- `tools/balance-lab/`：本地网页数值调参台。
- `assets/`：美术资源占位目录，当前主要用于后续 UI、鱼塘、特效和鱼类素材。
